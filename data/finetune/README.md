# Fine-Tuning Benchmark

Regex-to-DFA fine-tuning dataset introduced in Section 4.
Generated by `src/dataset/generate_dataset.py` with `seed=42`.

## Files

| File | Description |
|------|-------------|
| `regex_dfa_dataset.zip` | Full dataset — all splits, CoT format |
| `regex_dfa_dataset_train.zip` | Training split — ~22,500 examples |
| `regex_dfa_dataset_val.jsonl` | Validation split |
| `regex_dfa_dataset_test.jsonl` | Test split — ~1,000 examples |

## Unzip

```bash
cd data/finetune
unzip regex_dfa_dataset.zip
unzip regex_dfa_dataset_train.zip
```

## Tiers

| Tier | Share | DFA States | Description |
|------|-------|------------|-------------|
| 1 | ~15% | 2 | Direct symbol mapping |
| 2 | ~25% | 2–4 | Kleene star, plus, optional |
| 3 | ~35% | 3–6 | Alternation and branching |
| 4 | ~25% | 4–18 | Full NFA-to-DFA subset construction |

## Evaluation criterion (Appendix E.2.1)

Correct iff: parse succeeds + complete + deterministic + language-equivalent to ground truth
on all strings up to length 6. No partial credit.
